üöÄ Setup Instructions

Prerequisites

  • Node.js v16 or higher
  • A Fitbit account
  • A local LLM server (LM Studio, Ollama, etc.)

Step-by-Step Setup

  1. Clone the Repository 
    git clone https://github.com/MaxGer99/Ai-Health-tool.git
cd Ai-Health-tool
  2. Install Dependencies 
    npm install
  3. Register Fitbit Application

    Go to Fitbit Developer Portal and register a new app with:

    • Application Name: Your app name
    • Application Website: http://localhost:3001
    • Callback URL: http://localhost:3001/auth/fitbit/callback
    • OAuth 2.0 Type: Personal
    • Access Type: Read Only

    Copy your Client ID and Client Secret

  4. Setup Local LLM

    Install and run one of these:

    • LM Studio: Download from lmstudio.ai, load a model (Llama 3.1 recommended), and start the server
    • Ollama: Install from ollama.ai and run: ollama run llama3.1

    Default LLM server URL: http://localhost:1234/v1

  5. Configure Environment Variables 
    cp .env.example .env

    Edit .env with your credentials:

    FITBIT_CLIENT_ID=your_fitbit_client_id
FITBIT_CLIENT_SECRET=your_fitbit_client_secret
FITBIT_REDIRECT_URI=http://localhost:3001/auth/fitbit/callback

LLM_API_KEY=your_llm_api_key_here
LLM_API_URL=http://localhost:1234/v1
LLM_MODEL=local-model

PORT=3001
SESSION_SECRET=your_random_session_secret
  6. Run the Application 
    npm start

    Visit http://localhost:3001 in your browser

üéâ Success! You should now see the app running. Click "Connect with Fitbit" to authorize and start receiving AI coaching!

üîß API Endpoints

  • GET / - Main application page
  • GET /auth/fitbit - Initiate Fitbit OAuth flow
  • GET /auth/fitbit/callback - OAuth callback handler
  • GET /api/auth/status - Check authentication status
  • GET /api/fitbit/profile - Get user profile
  • GET /api/fitbit/activities - Get today's activities
  • POST /api/coach - Get AI coaching response
  • POST /api/auth/logout - Log out user

üêõ Troubleshooting

Fitbit OAuth Error

Make sure your callback URL in the Fitbit app settings matches exactly: http://localhost:3001/auth/fitbit/callback

LLM Not Responding

  • Ensure your local LLM server is running
  • Check the LLM_API_URL in your .env file
  • Verify the model is loaded in LM Studio/Ollama
  • Test the endpoint: curl http://localhost:1234/v1/models

No Data Displayed

  • Make sure you've granted all requested permissions to the Fitbit app
  • Check that you have recent activity data in your Fitbit account
  • Look at the browser console (F12) for error messages
  • Check the terminal where the server is running for backend errors

Port Already in Use

If port 3001 is already in use, change the PORT in your .env file and update the Fitbit callback URL accordingly.
